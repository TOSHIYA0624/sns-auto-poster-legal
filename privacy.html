Privacy Policy

Application: SNS Auto Poster
Effective date: June 5, 2026

1. Overview

SNS Auto Poster ("the App") is a personal, self-hosted automation tool operated by an individual ("the Operator") for the Operator's own use. This policy explains what data the App accesses and how it is used. The App has no public end users; it operates only on the Operator's own accounts.

2. Data the App Accesses

  • TikTok: Using the Content Posting API (scope video.publish), the App publishes the Operator's own video files to the Operator's own TikTok profile. If the Login Kit scope (user.info.basic) is granted, it is used only to identify the Operator's own account during authorization.
  • Google (Sheets / Drive / YouTube): The App reads a posting schedule from the Operator's own Google Sheet, reads the Operator's own video and thumbnail files from the Operator's own Google Drive, and uploads videos to the Operator's own YouTube channel.
  • Threads (Meta): Using the Threads API with permissions the Operator grants (threads_basic, threads_content_publish, threads_manage_replies, threads_keyword_search), the App accesses the Operator's own Threads profile ID and username and an access token, and—via keyword search—publicly available Threads posts that match keywords or hashtags the Operator chooses. For each public post this includes the post ID, text, permalink, author username, and timestamp. This data is used only to let the Operator review candidate posts and publish replies the Operator manually writes and approves.
  • Authentication tokens: OAuth refresh tokens for the connected accounts are stored locally in the Operator's own environment configuration and are used only to authenticate API requests.

3. How Data Is Used

Accessed data is used exclusively to schedule and publish the Operator's own content to the Operator's own social media accounts, and—for Threads—to let the Operator review publicly available posts and publish replies that the Operator manually writes and approves. The App does not use the data for any other purpose.

4. Data Sharing

The App does not sell, share, or transmit any accessed data to third parties. Data is sent only to the official APIs of the relevant platforms (Google, TikTok) for the sole purpose of publishing the Operator's own content.

5. Data Storage and Retention

The App runs locally and does not operate a public server or database. Downloaded video files are stored temporarily and deleted after each posting run. Access tokens are retained locally only for as long as needed to operate the App and can be revoked by the Operator at any time through the respective platform's account settings.

6. Threads Keyword Search and Public Post Data

The threads_keyword_search permission is used only to gather candidate public posts for the Operator to review and reply to. Publicly available posts retrieved via keyword search are stored solely in the Operator's private Google Spreadsheet alongside the Operator's reply records, are retained only as long as needed for the Operator's reply workflow, and can be deleted by the Operator at any time. This public post data is never sold, shared, redistributed, or used for profiling, advertising, or any purpose other than enabling the Operator's manual reply workflow. Access tokens are stored in the Operator's private Google Apps Script properties.

7. Third-Party Services

The App relies on services provided by Google, TikTok, and Meta (Threads). Their handling of data is governed by their own privacy policies.

8. Children's Privacy

The App is a personal tool not directed to children and does not knowingly collect data from children.

9. Changes

This Privacy Policy may be updated from time to time. Any updates will be posted on this page.
